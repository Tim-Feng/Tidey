Feature Highlights

Three reasons Tidey feels different.

These are the strongest differentiators in the codebase: workspaces that understand agents, an editor that lives beside the terminal, and a terminal core that keeps iTerm2 and tmux behavior intact.

01

Agent-aware workspaces

Each workspace carries its own terminal, title, status, and unread notification badge. Claude Code hooks can report Running, Idle, and Needs input directly into Tidey through the socket API, including inside tmux sessions.

Switch workspaces with ⌘1-⌘9, cycle with ⌃⌘] and ⌃⌘[, or jump back with ⌃⌘\.
02

Code stays beside the shell

Cmd-click a path in the terminal and open it in the built-in Monaco editor. Tidey keeps a file tree rooted to the current project, supports syntax highlighting, find, save, multiple editor tabs, and reloads clean external file changes.

The editor panel, terminal, sidebar, and file tree are independently collapsible and resizable.
03

iTerm2 depth, not a web shell

Tidey adds its own workspace and editor shell on top of iTerm2 instead of replacing the terminal core. The result keeps mature terminal emulation, native tmux -CC behavior, shell integration, and a layout that still feels like a real macOS app.

macOS 12+, Apple Silicon and Intel, built around the terminal first instead of treating it as an embed.

Keyboard Shortcuts

Fast enough to stay in flow.

The layout is built around keyboard movement: switch workspaces, reveal panels, hop between editor tabs, and save without leaving the terminal-first rhythm.

New workspace ⌘N
New panel ⌘T
Close current item ⌘W
Switch workspace ⌘1-⌘9
Next workspace ⌃⌘]
Previous workspace ⌃⌘[
Last workspace ⌃⌘\
Toggle sidebar ⌘B
Toggle editor ⇧⌘E
Toggle terminal ⇧⌘T
Toggle file tree ⌃⌘F
Panel or editor tab by number ⌃1-⌃9
Find in editor ⌘F
Save file ⌘S
Discover, don't display

Tidey can reveal numbered panel hints when Command is held, so the interface stays quiet until a shortcut is needed.

One window, separate rhythms

Workspace switching lives on the Command layer. Panel and editor tab switching lives on the Control layer, depending on what currently has focus.
